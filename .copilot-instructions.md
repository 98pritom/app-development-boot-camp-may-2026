# MExpense App - Copilot Instructions

## Project Overview

**MExpense** is a modern Flutter expense tracking application designed to help users monitor and analyze their spending patterns. The app provides a beautiful, intuitive interface for adding expenses, viewing summaries, and gaining insights into spending habits with visual analytics.

**Target Platform:** Cross-platform (iOS, Android, Web, Windows, macOS, Linux)  
**Language:** Dart 3.10.8+  
**Framework:** Flutter 3.x with Material Design 3  
**State Management:** Riverpod  
**Local Storage:** Hive

## Architecture & Design Patterns

### **Clean Architecture**
The project follows a clean, scalable architecture:
- **Models** (`core/models/`) - Data structures (Hive adapters for persistence)
- **Features** (`features/`) - Self-contained feature modules with views and view models
- **Shared** (`shared/`) - Reusable widgets and utilities across features
- **Core** (`core/`) - App-wide constants, theme, and models

### **State Management**
- **Riverpod** for reactive state management
- View Models handle feature-specific state
- Providers are immutable and declarative
- Example: `home_view_model.dart` manages tab selection state

### **Data Persistence**
- **Hive** for local storage with type-safe adapters
- Expense model is Hive-stored with auto-adapter generation
- Storage box is initialized in `main.dart`

### **UI/UX Framework**
- **Material Design 3** with Material You theming
- Modern gradient cards for visual hierarchy
- Responsive layouts with proper spacing and typography
- Smooth animations and transitions

## Coding Conventions

### **Naming**
- Classes: `PascalCase` (e.g., `ExpenseCard`)
- Functions/Variables: `camelCase` (e.g., `_buildEmptyState()`, `monthlyTotal`)
- Constants: `camelCase` (e.g., `expenseBoxName`)
- Private members: prefix with `_` (e.g., `_amountController`)

### **File Organization**
- Group related logic in dedicated directories
- Keep screens and view models separate
- Widgets in `widgets/` subdirectory
- One main class per file (with small helpers allowed)

### **Widget Structure**
```dart

class MyWidget extends StatelessWidget {
  const MyWidget({super.key, required this.property});
  
  final String property;
  
  @override
  Widget build(BuildContext context) {
    return /* ... */;
  }

// 3. Helper methods at the end
```

### **Style Guidelines**
- Use `const` for immutable widgets
- Always specify widget keys in constructors
- Use null-coalescing and null-safety operators
- Format with consistent indentation (2 spaces)
- Add meaningful comments for complex logic

## Best Practices for This Project

### **When Adding New Features**
1. Create feature folder in `features/`
2. Add `view/screens/`, `view/widgets/`, `view_model/` structure
3. Use Riverpod providers for state management
4. Keep business logic in view models, UI logic in widgets

### **When Adding UI Components**
1. Use Material 3 theme colors from `app_theme.dart`
2. Apply consistent spacing (8px, 12px, 16px, 20px, 24px units)
3. Add proper shadows using `Colors.black.withValues(alpha: 0.05)` style
4. Use `BorderRadius.circular(12)` for standard corners
5. Follow Material 3 elevation guidelines

### **When Modifying Expenses**
1. Always generate new UUIDs for new expenses
2. Validate dates against the 7-day window
3. Update Hive box after modifications
4. Use `ValueListenableBuilder` to react to storage changes

### **When Using Colors**
- Never use `withOpacity()` - use `withValues(alpha:)` instead
- Reference hex colors as `const Color(0xFFRRGGBB)`
- Use theme colors from `AppTheme` for consistency

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Hive adapter not found | Run `flutter clean`, rebuild |
| Date picker showing wrong range | Check `AppConstants.expenseDateRangeDays` |
| Colors not updating | Ensure `withValues(alpha:)` is used, not deprecated `withOpacity()` |
| Expenses not persisting | Verify `Hive.box()` is opened in `main.dart` |
| Riverpod state not updating | Check provider invalidation in view model methods |
